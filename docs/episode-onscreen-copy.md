# 上屏文案清单

## 1. 文档用途

这份文档是《不被预测的人生》这一集的第一版上屏文案制作稿。

它服务的对象包括：

- 内容设计
- 产品设计
- UI 设计
- 配音与数字人团队
- 前端开发

这份文档的目标不是解释剧情，而是明确：

- 哪些文字会上屏
- 在哪个场景上屏
- 由谁说
- 是常驻文字、按钮文案、提示文案，还是剧情台词

## 2. 使用原则

- 上屏文案必须短、清楚、可感知
- 不把抽象概念直接塞给孩子
- 不把重要信息全挤进一段长文
- 优先用角色、系统、界面和动作承载信息
- 所有文案都保持世界内语气

## 2.5 当前实现边界（2026-04-05）

这份上屏文案里，有一部分已经进了 current 主线，有一部分还只是后补候选。为避免页面把候选当上线，这一版统一按下面三层看：

- `已进 current`
  - 主线标题、副标题、按钮、关键台词
  - 礼仪辅助器的边角轻提示
  - 风名标签的基础层
- `文稿保留，但不算 current 完成`
  - 荒诞微未来整套插片
  - 礼仪辅助器和飞行器的大量轮换句
  - 更多可点击风声短音频
- `当前不进主线承诺`
  - 会打断剧情的大型收集层
  - 会把暗层变成小游戏的整套趣味包装

## 3. 文案类型说明

建议将本集上屏文案分成 7 类：

1. 场景标题文案
2. 系统状态文案

4. 角色关键台词
5. 世界趣味文案

### 5.11 `A05` 节奏补充

用于 `clean plate + UI 后叠` 的当前节奏建议。当前推荐总时长从 `5s` 放宽到约 `7.2s`，做成“召唤式展开 + 长停留呼吸”：

1. `0.0s - 0.8s`
   只保留灰银脉冲、顶部标签和很轻的场景状态，先让世界自己呼吸
2. `0.8s - 1.8s`
   中央标题与请求短句淡入，形成“任务已被唤起”的第一落点
3. `1.8s - 3.1s`
   左侧任务目标卡出现
4. `3.1s - 4.3s`
   右侧队伍状态卡点亮
5. `4.3s - 5.3s`
   倒计时补进，让责任感在后半拍落到玩家身上
6. `5.3s - 6.0s`
   阿潮只以远处轻掠过的光迹扫过右上一次，不抢画面中心，不做实体入画
7. `6.0s - 7.2s`
   底部按钮落下并轻微停稳，形成“现在你可以行动”的收束感

补充约束：

- 不要让倒计时从第 `0` 帧就像考试计时器一样压在画面上
- 不要做所有模块齐进齐出的机械循环
- 如果预览必须循环，循环感应尽量隐藏在长停留尾段，不要反复重演完整召唤动作

### 5.12 `A06` 航线点亮与启航提示

这一拍负责把 `接受航行` 从按钮动作变成真正启航，不再让 `A05` 同时承担任务简报和趣味节奏。

- 阿潮短播报建议：
  - `阿潮：航线亮了。别慌，我今天只负责飞稳，不替你下判断。`
  - 备选更克制版：`阿潮：入港航线开了。下一站，时间档案城。`
- 启航状态短文案：
  - `航线已确认`
  - `正在接入临时观察许可前置航道`
  - `时间档案城预计到达：00:42`

执行约束：

- 阿潮在 `A06` 里是执行型提示，不是搞笑打断
- 首版如果画面仍容易漂，优先只留声音和航线点亮，不强求实体入画
