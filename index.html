提示词反思与定稿策略

为什么之前迟迟不到位

  1. 我把“角色定义”直接当成了“生图提示词”。这会导致提示词里混进太多抽象概念，比如“制度成立”“公共表达”“温柔托管”，模型会自己脑补成它最熟悉的视觉套路。
  2. 我没有足够警惕模型的强先验。例如一旦写了 `officer / uniform / civic authority`，模型就会天然往军警、徽章、肩章、制服走。
  3. 我试图用一条 prompt 同时解决太多事。年龄、性格、服装、镜头、世界观、负向排除全塞在一起，结果真正最重要的约束没有成为最高优先级。
  4. 我对“禁止文字污染”的控制不够硬。只写“no text in frame”不够，凡是出现确认书、屏幕、档案墙、标牌，模型都会强烈想生成字。
  5. 我还在用“探索式试图找感觉”的方式工作，而不是像正式制作那样，先锁少量关键锚点，再围绕锚点做受控变体。

现在改成什么方法

  1. 把角色 prompt 分层：先锁脸和年龄，再锁服装，再锁动作，不再一条 prompt 什么都做。
  2. 把失败资产单独建模：像序列员 07 这种，我不再叫他 `officer`，而改成 `系统注册官 / 行政架构员` 这类反先验词。
  3. 场景分成“空间母图”和“界面/文字层”：需要确认书、档案墙、状态字的，不交给生图模型一次生成，而是后续单独叠 UI。
  4. 只做最少但有效的迭代：保留已经对的方向，不再把每一轮都从头推翻。
  5. 从现在开始以“定稿优先”而不是“试试看”优先：只修真正失败的点，不再同时扩太多新资产。

我前面最典型的三个错误

  • 序列员 07：我明明知道你要“制度感，不要军警感”，却还保留了会触发军装先验的词。
  • 场景母图：我让模型直接生成“确认书 / 档案墙 / 屏幕内容”，这几乎注定会带来乱码。
  • 闻夏和林澄：我虽然在调年龄，但没有把“12 岁骨相”放到真正最高优先级，所以模型不断往更小龄或更甜的方向偏。

Robot 伙伴判断

是否值得引入

结论：可以引入，但必须是“物理执行型伙伴”，不能成为第二个回响，也不能成为另一个阿潮。

  • 它应该补上：钻缝、取样、搬运、现场探测、实体互动。
  • 它不应该负责：讲解世界、替孩子思考、卖萌撑趣味。
  • 正确方向：低语音或几乎不说话，主要靠灯光、姿态、短提示音表达。

当前原型问题

  • `v1` 和 `v2` 都偏玩具化。
